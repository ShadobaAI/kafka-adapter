# Руководство пользователя

## API `кфкИнтеграция`

Модуль `кфкИнтеграция` — основная точка входа для прикладного разработчика. Вызывать из прикладного кода следует только этот модуль и `кфкИнтеграцияКлиент`.

### Отключение регистрации объекта

```bsl
// Источник — объект 1С (например, ДокументОбъект или СправочникОбъект)
кфкИнтеграция.Отключить(Источник);
```

- Помечает объект флагом, который предотвращает его регистрацию к обмену.
- Удобно использовать в сценариях, когда определённые изменения не должны уходить во внешние системы.

### Помещение данных в очередь исходящих сообщений

```bsl
// Отправка одного документа по ссылке
СсылкаНаДокумент = Документы.ЗаказПокупателя.НайтиПоНомеру(Номер);
кфкИнтеграция.ПоместитьВОчередьИсходящих(СсылкаНаДокумент);

// Отправка массива ссылок
МассивСообщений = Новый Массив;
МассивСообщений.Добавить(СсылкаНаДокумент1);
МассивСообщений.Добавить(СсылкаНаДокумент2);
кфкИнтеграция.ПоместитьВОчередьИсходящих(МассивСообщений);
```

Поддерживаемые варианты параметра `Источник`:
- ссылка на объект (справочник, документ и т.п.);
- объект (`СправочникОбъект`, `ДокументОбъект`, набор записей регистра сведений и др.);
- `РегистрСведенийНаборЗаписей`;
- `Структура` / `Соответствие` с произвольным содержимым;
- результат `кфкИнтеграция.ПараметрыСообщения()` — готовое сообщение с телом, ключом и заголовками (сериализация пропускается);
- массив из перечисленных типов.

Дополнительные параметры:
- `ТипОбъект` — строка с обозначением типа объекта для использования в обработчиках;
- `Продюсеры` — `СправочникСсылка.кфкПродюсеры` или массив/коллекция продюсеров для маршрутизации.

Адаптер поддерживает пакетный режим обработки при передаче массива: сообщения разбиваются на порции, регистрация завершается порционно.

## Контракты обработчиков

Для каждого продюсера и консьюмера выбирается способ обработки сообщений:

- **Конвертация данных** — преобразование через правила 1С:Конвертации данных 3.1; потребуется XDTO-пакет и модуль обмена, сгенерированный в КД 3.1.
- **Произвольный обработчик** — экспортный метод общего модуля прикладного решения. Имя модуля и метода указываются в табличной части продюсера или консьюмера.

### Обработчик продюсера

Формирует тело сообщения для отправки в Kafka:

```bsl
Функция Сериализация(Объект, Свойства, ТекстЖурнала, Отказ) Экспорт
```

**Параметры:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `Объект` | `СправочникОбъект`, `ДокументОбъект`, `ЛюбаяСсылка`, `Структура`, `Соответствие` | Регистрируемый объект. При регистрации по событию записи — объект или набор записей регистра; при регистрации через `кфкИнтеграция.ПоместитьВОчередьИсходящих()` — переданное значение. Если при регистрации набора записей регистра набор пуст, флаг `ЭтоУдаление` в `Свойства` будет `Истина`, а `Объект` будет содержать значения полей отбора. |
| `Свойства` | `Структура` | Дополнительные свойства сообщения (см. ниже) |
| `ТекстЖурнала` | `Строка` | Текст, который будет записан в журнал регистрации |
| `Отказ` | `Булево` | Установите в `Истина` для отмены формирования и отправки сообщения |

**Поля структуры `Свойства`:**

| Поле | Тип | Описание |
|------|-----|----------|
| `Продюсер` | `СправочникСсылка.кфкПродюсеры` | Продюсер сообщения |
| `ТипОбъект` | `Строка` | Тип объекта, к которому относится сообщение |
| `ТипИсходныхДанных` | `Строка` | Тип исходных данных |
| `СобытиеРегистрации` | `ПеречислениеСсылка.кфкСобытияРегистрации` | Способ регистрации. Для проверки используйте: `кфкИнтеграция.ЭтоРегистрацияПриЗаписи(Свойства)`, `кфкИнтеграция.ЭтоРегистрацияПринудительно(Свойства)`, `кфкИнтеграция.ЭтоРегистрацияПрограммно(Свойства)` |
| `КлючРегистрации` | `Строка` | Уникальный ключ регистрации |
| `ДатаРегистрации` | `Дата` | Дата и время регистрации события |
| `ДатаРегистрацииМС` | `Число` | Дата регистрации в миллисекундах |
| `ПоследняяОбработка` | `Дата` | Дата последней обработки сообщения |
| `ЭтоУдаление` | `Булево` | `Истина`, если регистрация вызвана событием удаления |
| `СсылкаНаОбъект` | `ЛюбаяСсылка`, `Неопределено` | Ссылка на объект регистрации |

**Возвращаемое значение** — результат `кфкИнтеграция.ПараметрыСообщения()` или `Неопределено` (сообщение не будет отправлено):

| Поле | Тип | Описание |
|------|-----|----------|
| `Данные` | `Строка`, `ДвоичныеДанные`, `Структура`, `Массив Из Структура` | Тело сообщения |
| `Ключ` | `Строка` | Уникальный идентификатор сообщения |
| `Заголовки` | `Соответствие Из Строка` | Заголовки сообщения |

### Обработчик консьюмера

Обрабатывает тело сообщения, полученного из Kafka:

```bsl
Процедура Десериализация(ТелоСообщения, Свойства, ТекстЖурнала, Отказ) Экспорт
```

**Параметры:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `ТелоСообщения` | `Строка`, `ДвоичныеДанные` | Тело сообщения из Kafka. Тип зависит от настройки **Двоичные данные** в консьюмере |
| `Свойства` | `Структура` | Дополнительные свойства сообщения (см. ниже) |
| `ТекстЖурнала` | `Строка` | Текст, который будет записан в журнал регистрации |
| `Отказ` | `Булево` | Установите в `Истина` для отмены обработки сообщения |

**Поля структуры `Свойства`:**

| Поле | Тип | Описание |
|------|-----|----------|
| `Консьюмер` | `СправочникСсылка.кфкКонсьюмеры` | Консьюмер, получивший сообщение |
| `Топик` | `Строка` | Имя топика |
| `Ключ` | `Строка` | Ключ (идентификатор) сообщения, используемый для партиционирования |
| `Раздел` | `Строка` | Номер раздела (partition) |
| `Смещение` | `Строка` | Смещение (offset) сообщения в разделе |
| `Заголовки` | `Соответствие Из Строка` | Заголовки сообщения |
| `ДатаЗагрузки` | `Дата` | Дата получения сообщения системой |
| `ДатаОбработки` | `Дата` | Дата последней обработки сообщения |

## Сценарии использования (исходящие сообщения)

### Основной сценарий — автоматическая регистрация через подписки

Код прикладного решения не требуется. Достаточно настройки:

1. В справочнике **Продюсеры** добавьте имя типа метаданных в табличную часть **Объекты метаданных** (например, `Документ.РеализацияТоваровУслуг`).
2. Адаптер уже подписан на события записи и удаления всех ссылочных объектов и наборов записей регистров — при записи объект автоматически попадает в очередь и уходит в Kafka.

Никаких изменений в модулях прикладного решения не нужно.

### Дополнительные API-сценарии

Используйте `кфкИнтеграция` когда нужно выйти за рамки автоматической регистрации.

#### Исключение объекта из регистрации

Если определённые изменения не должны уходить во внешние системы — подавите регистрацию в обработчике `ПередЗаписью`:

```bsl
Если ЭтоТестовыйДокумент Тогда
    кфкИнтеграция.Отключить(ЭтотОбъект);
КонецЕсли;
```

#### Принудительная регистрация

Для явной постановки объекта в очередь в обход подписок — например, при повторной выгрузке или нестандартном сценарии:

```bsl
кфкИнтеграция.ПоместитьВОчередьИсходящих(СсылкаНаДокумент);
```

#### Пакетная регистрация

Для массовых операций (импорт, миграция, пересчёт):

```bsl
МассивОбъектов = Новый Массив;

Для Каждого СтрокаТаблицы Из ТаблицаДанных Цикл
    Dok = Документы.ЗаказПокупателя.СоздатьДокумент();
    // ... заполнение реквизитов ...
    Dok.Записать();
    МассивОбъектов.Добавить(Dok.Ссылка);
КонецЦикла;

кфкИнтеграция.ПоместитьВОчередьИсходящих(МассивОбъектов);
```

Адаптер разделит массив на порции и выполнит порционную регистрацию.

#### Отправка произвольных сообщений

Если требуется отправить данные, не связанные с конкретным объектом 1С:

```bsl
Сообщение = кфкИнтеграция.ПараметрыСообщения(СтрокаJSON, "Order-123");
// Сообщение.Заголовки.Вставить("source", "crm"); // при необходимости

кфкИнтеграция.ПоместитьВОчередьИсходящих(Сообщение);
```

## Работа с входящими сообщениями

Логика обработки входящих сообщений реализуется в обработчике консьюмера (см. раздел [Контракты обработчиков](#контракты-обработчиков) выше).

Рекомендации:
- разделяйте обработчики по типам сообщений или топикам;
- обеспечьте идемпотентность обработки — возможность безопасного повторного запуска без побочных эффектов.

## Прямой API (без очереди)

Используйте низкоуровневый API, когда нужна **синхронная** отправка или чтение без участия очереди и фоновых заданий.

### Отправка

```bsl
// Создать описание сообщения
Сообщение = кфкИнтеграция.ПараметрыСообщения("{ ""key"": ""value"" }", "идентификатор-123");

// Отправить в топик
Результат = кфкИнтеграция.Отправить(Продюсер, "имя-топика", Сообщение);

Если Результат.Успешно Тогда
    // сообщение доставлено
Иначе
    // Результат.ТекстОшибки
КонецЕсли;
```

Для **пакетной отправки** с удержанием соединения:

```bsl
Сессии = Новый Соответствие;

Для Каждого Сообщение Из СписокСообщений Цикл
    кфкИнтеграция.Отправить(Продюсер, "имя-топика", Сообщение, 1, Сессии);
КонецЦикла;

кфкИнтеграция.ЗакрытьСессииПродюсера(Сессии);
```

### Чтение

```bsl
Сессии = Новый Соответствие;

// АвтоФиксацияСмещения = Истина — смещение фиксируется автоматически после чтения.
// АвтоФиксацияСмещения = Ложь  — необходимо вручную вызвать кфкИнтеграция.ЗафиксироватьСмещение().
Результат = кфкИнтеграция.Прочитать(Консьюмер, Истина, Сессии);

Если Результат.Успешно И Результат.Сообщение <> Неопределено Тогда
    Тело     = Результат.Сообщение.Тело;
    Ключ     = Результат.Сообщение.Ключ;
    Топик    = Результат.Сообщение.Топик;
    Смещение = Результат.Сообщение.Смещение;
КонецЕсли;

кфкИнтеграция.ЗакрытьСессииКонсьюмера(Сессии);
```
