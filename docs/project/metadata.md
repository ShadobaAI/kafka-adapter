# Объекты метаданных

Полный перечень объектов метаданных подсистемы (префикс **`кфк`**).

## Справочники

| Справочник | Назначение |
|------------|-----------|
| `кфкБрокеры` | Параметры подключения к кластерам Kafka |
| `кфкПродюсеры` | Настройки публикации сообщений (топик, тип обработчика, фильтр объектов) |
| `кфкКонсьюмеры` | Настройки загрузки сообщений (топик, тип обработчика, группа потребителей) |
| `Пользователи` | **Только для разработки** — обеспечивает ссылочную целостность в EDT; из финального билда **удаляется** |

## Регистры сведений

### Очереди сообщений

| Регистр | Назначение |
|---------|-----------|
| `кфкИсходящиеСообщения` | Очередь сообщений для отправки из 1С в Kafka |
| `кфкВходящиеСообщения` | Очередь сообщений, полученных из Kafka для обработки в 1С |

### Служебные регистры

| Регистр | Назначение |
|---------|-----------|
| `кфкОчередьПотоков` | Распределение сообщений между потоками обработки |
| `кфкПозицииОпераций` | Состояние обработки сообщений в разрезе фоновых операций |
| `кфкПараметрыКонтроляИнтеграции` | Настройки алертов и контрольных порогов |
| `кфкПараметрыЛогирования` | Настройки выгрузки истории обмена во внешний сервис логирования (ELK) |
| `кфкПараметрыОчисткиХранилища` | Настройки автоматического удаления устаревших записей из очередей, включая срок хранения хэш-сумм сообщений (по умолчанию 3 дня) |
| `кфкХэшСуммыСообщений` | История SHA-256-хэшей тел сообщений для [идемпотентной отправки и обработки](../user/configuration/producers.md#идемпотентная-отправка). Измерение — ключ записи очереди, ресурс — хэш-сумма; очищается по сроку хранения регламентным заданием |

## Перечисления

| Перечисление | Назначение |
|--------------|-----------|
| `кфкСтатусыСообщений` | Статусы записей в очередях: **Новое**, **Обработано**, **ОбработкаОтменена**, **ОшибкаОбработки**, **Выгружено**, **ОшибкаВыгрузки**, **ВыгрузкаОтменена** |
| `кфкТипыОбработчиков` | Тип обработчика сообщений: **КД 3.1** или **Произвольный** |
| `кфкСобытияРегистрации` | Способ постановки в очередь: **при записи**, **программно**, **принудительно** |

## Функциональные опции

Все опции работают в привилегированном режиме получения (`privilegedGetMode`).

| Опция | Хранилище значения | Назначение |
|-------|--------------------|-----------|
| `кфкИспользоватьИнтеграцию` | Константа `кфкИспользоватьИнтеграцию` | Главная опция: включает подсистему и управляет видимостью практически всех её объектов в интерфейсе |
| `кфкБлокировкаИнтеграции` | Константа `кфкБлокировкаИнтеграции` | Программный флаг временной блокировки обмена (состав объектов пуст) |
| `кфкИспользоватьКонтрольИнтеграции` | Константа `кфкИспользоватьКонтрольИнтеграции` | Видимость объектов контроля: регистры `кфкПараметрыКонтроляИнтеграции`, `кфкПозицииОпераций` |
| `кфкИспользоватьПотоковыйРежим` | Константа `кфкИспользоватьПотоковыйРежим` | Программный флаг потокового режима (состав объектов пуст) |
| `кфкИспользоватьВнешнееЛогирование` | Ресурс `Использовать` регистра `кфкПараметрыЛогирования` | Программный флаг выгрузки журнала обмена во внешний логгер; единственная опция, основанная не на константе |

## Роли

| Роль | Назначение |
|------|-----------|
| `кфкБазовыеПрава` | Наблюдатель: чтение настроек, очередей и истории обмена без права изменения |
| `кфкПолныеПрава` | Администратор адаптера: полный доступ к настройкам, константам и регистрации изменений |

Подробное описание — [Роли и права](../user/installation/roles.md).

## Обработки

| Обработка | Назначение |
|-----------|-----------|
| `кфкИнтеграция` | Низкоуровневая работа с внешней компонентой (DLL), фасад над Simple Kafka Connector 1C |
| `кфкПанельАдминистрирования` | UI-интерфейс управления и мониторинга |
| `кфкРегистрацияИзменений` | Принудительная постановка данных в очередь через UI |

## Регламентное задание

| Задание | Назначение |
|---------|-----------|
| `кфкЗапускИнтеграции` | Запускает диспетчер обработки: инициирует потоки сериализации, выгрузки, загрузки и десериализации |

## Определяемый тип

| Тип | Назначение |
|-----|------------|
| `кфкПользователь` | Ссылка на пользователя целевой конфигурации. По умолчанию указывает на служебный справочник `Пользователи` из состава адаптера; при внедрении **заменяется** на справочник пользователей прикладной конфигурации — см. [Адаптация подсистемы](../user/installation/adaptation.md) |
