# Объекты метаданных

Полный перечень объектов метаданных подсистемы (префикс **`кфк`**).

## Справочники

| Справочник | Назначение |
|------------|-----------|
| `кфкБрокеры` | Параметры подключения к кластерам Kafka |
| `кфкПродюсеры` | Настройки публикации сообщений (топик, тип обработчика, фильтр объектов) |
| `кфкКонсьюмеры` | Настройки загрузки сообщений (топик, тип обработчика, группа потребителей) |
| `Пользователи` | **Только для разработки** — обеспечивает ссылочную целостность в EDT; из финального билда **удаляется** |

## Регистры сведений

### Очереди сообщений

| Регистр | Назначение |
|---------|-----------|
| `кфкИсходящиеСообщения` | Очередь сообщений для отправки из 1С в Kafka |
| `кфкВходящиеСообщения` | Очередь сообщений, полученных из Kafka для обработки в 1С |

### Служебные регистры

| Регистр | Назначение |
|---------|-----------|
| `кфкОчередьПотоков` | Распределение сообщений между потоками обработки |
| `кфкПозицииОпераций` | Состояние обработки сообщений в разрезе фоновых операций |
| `кфкПараметрыКонтроляИнтеграции` | Настройки алертов и контрольных порогов |
| `кфкПараметрыЛогирования` | Настройки выгрузки истории обмена во внешний сервис логирования (ELK) |
| `кфкПараметрыОчисткиХранилища` | Настройки автоматического удаления устаревших записей из очередей |

## Перечисления

| Перечисление | Назначение |
|--------------|-----------|
| `кфкСтатусыСообщений` | Статусы записей в очередях: **Новое**, **Обработано**, **ОбработкаОтменена**, **ОшибкаОбработки**, **Выгружено**, **ОшибкаВыгрузки** |
| `кфкТипыОбработчиков` | Тип обработчика сообщений: **КД 3.1** или **Произвольный** |
| `кфкСобытияРегистрации` | Способ постановки в очередь: **при записи**, **программно**, **принудительно** |

## Обработки

| Обработка | Назначение |
|-----------|-----------|
| `кфкИнтеграция` | Низкоуровневая работа с внешней компонентой (DLL), фасад над Simple Kafka Connector 1C |
| `кфкПанельАдминистрирования` | UI-интерфейс управления и мониторинга |
| `кфкРегистрацияИзменений` | Принудительная постановка данных в очередь через UI |

## Регламентное задание

| Задание | Назначение |
|---------|-----------|
| `кфкЗапускИнтеграции` | Запускает диспетчер обработки: инициирует потоки сериализации, выгрузки, загрузки и десериализации |

## Определяемый тип

| Тип | Назначение |
|-----|------------|
| `кфкПользователь` | Ссылка на пользователя целевой конфигурации. По умолчанию указывает на служебный справочник `Пользователи` из состава адаптера; при внедрении **заменяется** на справочник пользователей прикладной конфигурации — см. [Настройка в EDT](../user/installation/edt-setup.md) |
