# Управляющие константы

Константы, управляющие поведением подсистемы (префикс **`кфк`**).

| Константа | Назначение |
|-----------|-----------|
| `кфкИспользоватьИнтеграцию` | Глобальное включение/выключение подсистемы |
| `кфкБлокировкаИнтеграции` | Временная приостановка обмена без отключения подсистемы |
| `кфкИспользоватьКонтрольИнтеграции` | Включение мониторинга и алертов |
| `кфкИспользоватьПотоковыйРежим` | Включение многопоточного режима обработки |
| `кфкКаталогЛогированияКомпоненты` | Путь на сервере для лог-файлов внешней компоненты |
| `кфкФорматИмениФайловЛогированияКомпоненты` | Шаблон имени лог-файла компоненты |

## Как управлять

Большинство констант меняются **не напрямую**, а через панель администрирования:

| Константа | Где управляется |
|-----------|-----------------|
| `кфкИспользоватьИнтеграцию` | [Kafka / Администрирование → Включить подсистему](../user/configuration/index.md#включение-подсистемы) |
| `кфкБлокировкаИнтеграции` | [Kafka / Администрирование / Фоновые задания → Заблокировать / Разблокировать](../user/configuration/jobs.md#фоновые-задания) |
| `кфкИспользоватьКонтрольИнтеграции` | [Алерты и контроль → Использовать](../user/configuration/alerts.md#контроль-интеграции) |
| `кфкИспользоватьПотоковыйРежим` | [Фоновые и регламентные задания → Потоковый режим](../user/configuration/jobs.md#потоковый-режим) |
| `кфкКаталогЛогированияКомпоненты`, `кфкФорматИмениФайловЛогированияКомпоненты` | [Логирование → Каталог / Файлы](../user/configuration/logging.md#логирование-компоненты-kafka) |

!!! warning "Изменение напрямую"
    Прямая запись в константы **не рекомендуется** — панель администрирования выполняет дополнительные проверки и настройки при изменении этих значений (например, блокирует запуск фоновых заданий на время переключения).
