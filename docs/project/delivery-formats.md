# Форматы поставки

Подсистема поставляется в нескольких форматах — каждый для своей задачи.

## Как расширение конфигурации

| Файл | Назначение |
|------|------------|
| `Adapter Kafka v*.cfe` | Файл расширения — подключается через «Расширения конфигурации» в Конфигураторе |
| `Adapter Kafka v* (CFE).zip` | Проект EDT расширения — для разработки и доработки |
| `Adapter Kafka v*.xlib` | [Библиотека расширения для EDT](https://its.1c.ru/db/edtdoc#content:10626:hdoc) — для подключения и обновления проекта из библиотеки |

## Как часть основной конфигурации

| Файл | Назначение |
|------|------------|
| `Adapter Kafka v*.cf` | Файл поставки конфигурации — для переноса через стандартный механизм поставки 1С |
| `Adapter Kafka v* (CF).zip` | Проект EDT конфигурации — для разработки и доработки |
| `Adapter Kafka v*.clib` | [Библиотека расширения для EDT](https://its.1c.ru/db/edtdoc#content:10626:hdoc) — для подключения и обновления проекта из библиотеки |

## Демо-пример

| Файл | Назначение |
|------|-----------|
| `Adapter Kafka (ДЕМО) v*.dt` | Выгрузка информационной базы с настроенным примером интеграции — загружается через Конфигуратор |

## Что выбирать для разработки

- **Разработка адаптера** — работа напрямую с исходниками в каталоге `src/` через EDT.
- **Сборка релиза** — выгрузка в соответствующий формат через скрипты CI.
- **Прикладной разработчик, использующий адаптер** — см. [Установка и подключение](../user/installation/index.md) — выбирает между CFE/XLIB или CF/CLIB.

## Работа через библиотеки в EDT

Форматы `.xlib` и `.clib` — библиотечные проекты. Позволяют:

- Подключать адаптер как **зависимость** к прикладному проекту.
- Получать **обновления** из репозитория без ручного слияния изменений.
- **Переиспользовать** подсистему в нескольких проектах.

Подробнее: [1С:EDT — Работа с библиотеками](https://its.1c.ru/db/edtdoc#content:10626:hdoc).
