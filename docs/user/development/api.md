# Программный API

Модуль `кфкИнтеграция` — основная точка входа для прикладного разработчика. Вызывать из прикладного кода следует **только** этот модуль и `кфкИнтеграцияКлиент`.

## Обзор методов

| Метод | Назначение |
|-------|-----------|
| `ПоместитьВОчередьИсходящих(Источник, ТипОбъект, Продюсеры)` | Регистрирует объект/массив в очереди исходящих |
| `Отключить(Источник)` | Помечает объект флагом, предотвращающим регистрацию при записи |
| `ПараметрыСообщения(Данные, Ключ)` | Формирует описание сообщения: тело, ключ, заголовки |
| `Отправить(Продюсер, Топик, Сообщение, ..., Сессии)` | Синхронная отправка (прямой API) |
| `Прочитать(Консьюмер, АвтоФиксацияСмещения, Сессии)` | Синхронное чтение (прямой API) |
| `ЗакрытьСессииПродюсера(Сессии)` | Закрывает открытые сессии продюсера |
| `ЗакрытьСессииКонсьюмера(Сессии)` | Закрывает открытые сессии консьюмера |
| `ЗафиксироватьСмещение(Консьюмер, Сообщение, Сессии)` | Ручная фиксация смещения после чтения |

## Отключение регистрации объекта

```bsl
// Источник — объект 1С (например, ДокументОбъект или СправочникОбъект)
кфкИнтеграция.Отключить(Источник);
```

Помечает объект флагом, который предотвращает его регистрацию к обмену. Полезно, когда определённые изменения не должны уходить во внешние системы.

!!! warning "Только до записи"
    Вызывайте `Отключить()` **до** записи объекта — в обработчике `ПередЗаписью` или непосредственно перед вызовом `Записать()`. После записи вызов не имеет эффекта.

## Помещение данных в очередь исходящих

```bsl
// Отправка одного документа по ссылке
СсылкаНаДокумент = Документы.ЗаказПокупателя.НайтиПоНомеру(Номер);
кфкИнтеграция.ПоместитьВОчередьИсходящих(СсылкаНаДокумент);

// Отправка массива ссылок
МассивСообщений = Новый Массив;
МассивСообщений.Добавить(СсылкаНаДокумент1);
МассивСообщений.Добавить(СсылкаНаДокумент2);
кфкИнтеграция.ПоместитьВОчередьИсходящих(МассивСообщений);
```

### Поддерживаемые варианты параметра `Источник`

- ссылка на объект (справочник, документ и т.п.);
- объект (`СправочникОбъект`, `ДокументОбъект`, набор записей регистра сведений и др.);
- `РегистрСведенийНаборЗаписей`;
- `Структура` / `Соответствие` с произвольным содержимым;
- результат `кфкИнтеграция.ПараметрыСообщения()` — **готовое** сообщение с телом, ключом и заголовками (сериализация пропускается);
- массив из перечисленных типов.

### Дополнительные параметры

| Параметр | Назначение |
|----------|-----------|
| `ТипОбъект` | Строка — **ключ регистрации**. Адаптер ищет продюсер, у которого значение в «Объектах метаданных» **точно совпадает** с этой строкой. Может быть полным именем метаданных (`Справочник.Номенклатура`) или произвольным ключом (`crm.order-placed`). Подробно — [Тип объект — ключ регистрации](../configuration/producers.md#тип-объект-ключ-регистрации). |
| `Продюсеры` | `СправочникСсылка.кфкПродюсеры` или массив/коллекция продюсеров для маршрутизации |

Адаптер поддерживает **пакетный режим** обработки при передаче массива: сообщения разбиваются на порции, регистрация завершается порционно.

## Формирование «готового» сообщения

Когда нужно отправить произвольные данные (не связанные с объектом 1С):

```bsl
Сообщение = кфкИнтеграция.ПараметрыСообщения(СтрокаJSON, "Order-123");
Сообщение.Заголовки.Вставить("source", "crm");

кфкИнтеграция.ПоместитьВОчередьИсходящих(Сообщение);
```

Структура возврата `ПараметрыСообщения()`:

| Поле | Тип | Описание |
|------|-----|----------|
| `Данные` | `Строка`, `ДвоичныеДанные`, `Структура`, `Массив Из Структура` | Тело сообщения |
| `Ключ` | `Строка` | Уникальный идентификатор сообщения |
| `Заголовки` | `Соответствие Из Строка` | Заголовки сообщения |

## Вспомогательные методы

### Проверка способа регистрации

В обработчике продюсера часто полезно знать, **как** сообщение попало в очередь — через подписку, программно или принудительно:

```bsl
Если кфкИнтеграция.ЭтоРегистрацияПриЗаписи(Свойства) Тогда
    // Зарегистрировано подпиской на событие записи
КонецЕсли;

Если кфкИнтеграция.ЭтоРегистрацияПринудительно(Свойства) Тогда
    // Зарегистрировано вручную через UI «Регистрация изменений»
КонецЕсли;

Если кфкИнтеграция.ЭтоРегистрацияПрограммно(Свойства) Тогда
    // Зарегистрировано через API ПоместитьВОчередьИсходящих
КонецЕсли;
```
