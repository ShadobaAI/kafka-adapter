# Прямой API

!!! note "Когда использовать"
    Прямой API подходит для **синхронных** операций — когда нужен немедленный ответ о доставке или необходимо прочитать конкретное сообщение **без** участия фоновых заданий. Для регулярного обмена данными используйте очередь.

Используйте низкоуровневый API, когда нужна **синхронная** отправка или чтение без участия очереди и фоновых заданий адаптера.

## Отправка — один раз

```bsl
// Создать описание сообщения
Сообщение = кфкИнтеграция.ПараметрыСообщения("{ ""key"": ""value"" }", "идентификатор-123");

// Отправить в топик
Результат = кфкИнтеграция.Отправить(Продюсер, "имя-топика", Сообщение);

Если Результат.Успешно Тогда
    // сообщение доставлено
Иначе
    // Результат.ТекстОшибки
КонецЕсли;
```

## Отправка — пакетом (с удержанием соединения)

При пакетной отправке держите соединение открытым через **сессии**:

```bsl
Сессии = Новый Соответствие;

Попытка
    Для Каждого Сообщение Из СписокСообщений Цикл
        кфкИнтеграция.Отправить(Продюсер, "имя-топика", Сообщение, 1, Сессии);
    КонецЦикла;
Исключение
    кфкИнтеграция.ЗакрытьСессииПродюсера(Сессии);
    ВызватьИсключение;
КонецПопытки;

кфкИнтеграция.ЗакрытьСессииПродюсера(Сессии);
```

!!! warning "Всегда закрывайте сессии"
    Вызывайте `ЗакрытьСессииПродюсера` в том числе в блоке исключения — иначе соединение **зависнет** и будет удерживать ресурсы на сервере 1С.

## Чтение

```bsl
Сессии = Новый Соответствие;

// АвтоФиксацияСмещения = Истина  — смещение фиксируется автоматически после чтения.
// АвтоФиксацияСмещения = Ложь    — нужен ручной вызов кфкИнтеграция.ЗафиксироватьСмещение().
Результат = кфкИнтеграция.Прочитать(Консьюмер, Истина, Сессии);

Если Результат.Успешно И Результат.Сообщение <> Неопределено Тогда
    Тело     = Результат.Сообщение.Тело;
    Ключ     = Результат.Сообщение.Ключ;
    Топик    = Результат.Сообщение.Топик;
    Смещение = Результат.Сообщение.Смещение;
КонецЕсли;

кфкИнтеграция.ЗакрытьСессииКонсьюмера(Сессии);
```

### Ручная фиксация смещения

Если нужна гарантия «at-least-once» с явной фиксацией после успешной обработки:

```bsl
Сессии = Новый Соответствие;
Результат = кфкИнтеграция.Прочитать(Консьюмер, Ложь, Сессии);

Если Результат.Успешно И Результат.Сообщение <> Неопределено Тогда
    Попытка
        ОбработатьСообщение(Результат.Сообщение);
        // Фиксируем только после успешной обработки
        кфкИнтеграция.ЗафиксироватьСмещение(Консьюмер, Результат.Сообщение, Сессии);
    Исключение
        // Смещение не зафиксировано — сообщение будет прочитано повторно
        кфкИнтеграция.ЗакрытьСессииКонсьюмера(Сессии);
        ВызватьИсключение;
    КонецПопытки;
КонецЕсли;

кфкИнтеграция.ЗакрытьСессииКонсьюмера(Сессии);
```

## Когда стоит использовать прямой API

- **Синхронная отправка** команды во внешнюю систему с ожиданием ответа о доставке.
- **Чтение одного сообщения** (например, для проверки работоспособности кластера).
- **Периодические утилиты**, которым не нужна инфраструктура очередей.
- **Интеграционные тесты** — детерминированная отправка и чтение.

## Когда **не** стоит

- Регулярный массовый обмен — используйте очередь.
- Сценарии, где важна **устойчивость к сбоям**: прямой API не переживает перезапуски и не поддерживает автоматический повтор.
- Когда требуется **параллельная обработка** — очередь и диспетчеры уже это обеспечивают.
