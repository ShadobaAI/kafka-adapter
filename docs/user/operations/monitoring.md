# Мониторинг очередей

## Основные инструменты

| Инструмент | Где открыть | Назначение |
|-----------|------------|-----------|
| **Фоновые задания** | Kafka / Администрирование / Фоновые задания | Состояние потоков, ошибки фоновых заданий |
| **Регламентное задание** | Kafka / Администрирование / Регламентное задание | Мониторинг выполнения регламентного задания |
| **РС «Исходящие сообщения»** | Kafka / Исходящие сообщения | Детальный просмотр очереди исходящих, ручной перезапуск |
| **РС «Входящие сообщения»** | Kafka / Входящие сообщения | Детальный просмотр очереди входящих, ручной перезапуск |

## Что смотреть каждый день

1. **Размер очередей** — нет ли постоянно растущего числа сообщений в статусах **Новое** / **Ошибка**.
2. **Состояние фоновых заданий** — все ли потоки работают.
3. **Регламентное задание** — не упало ли, выполняется ли по расписанию.
4. **Уведомления в Telegram** (если включён [контроль интеграции](../configuration/alerts.md)) — есть ли нарушения SLA.

## Что смотреть еженедельно

1. **Накопление ошибок** — сколько сообщений в статусах «Ошибка выгрузки» и «Ошибка обработки».
2. **Журнал регистрации 1С** — необработанные исключения в фоновых заданиях.
3. **Сроки хранения** сообщений — не разрастается ли размер регистров сверх разумного.

## Автоматизированный мониторинг

Для продуктивного контура настройте:

- **[Алерты и контроль интеграции](../configuration/alerts.md)** — уведомления в Telegram при нарушении SLA.
- **[Внешнее логирование](../configuration/logging.md)** — выгрузка журнала обмена в ELK / Loki. В Kibana / Grafana можно построить дашборды.

## Что делать при росте очереди

Подробности — в [Типовые проблемы → Очередь растёт](troubleshooting.md#очередь-постоянно-растёт).

Краткая формула:

1. Увеличьте **подпотоки обработки** и **подпотоки транспорта** в диспетчере задач продюсера/консьюмера.
2. Включите **потоковый режим** регламентного задания.
3. Разделите нагрузку между **несколькими диспетчерами** или топиками.

## Примеры запросов для мониторинга

Готовые шаблоны запросов 1С, которые можно выполнять в консоли запросов, обработках-отчётах или регламентных проверках.

### Количество сообщений по статусам

```bsl
Запрос = Новый Запрос;
Запрос.Текст =
    "ВЫБРАТЬ
    |   ИсходящиеСообщения.Статус КАК Статус,
    |   КОЛИЧЕСТВО(*)             КАК Количество
    |ИЗ
    |   РегистрСведений.кфкИсходящиеСообщения КАК ИсходящиеСообщения
    |СГРУППИРОВАТЬ ПО
    |   ИсходящиеСообщения.Статус";

Выборка = Запрос.Выполнить().Выбрать();
Пока Выборка.Следующий() Цикл
    Сообщить(Выборка.Статус + ": " + Выборка.Количество);
КонецЦикла;
```

### Сообщения со статусом «Ошибка выгрузки», ожидающие повтора

```bsl
Запрос = Новый Запрос;
Запрос.Текст =
    "ВЫБРАТЬ
    |   ИсходящиеСообщения.КлючРегистрации КАК КлючРегистрации,
    |   ИсходящиеСообщения.Продюсер        КАК Продюсер,
    |   ИсходящиеСообщения.КоличествоВыгрузок КАК Попыток,
    |   ИсходящиеСообщения.ТекстЖурнала    КАК Ошибка
    |ИЗ
    |   РегистрСведений.кфкИсходящиеСообщения КАК ИсходящиеСообщения
    |ГДЕ
    |   ИсходящиеСообщения.Статус = &ОшибкаВыгрузки
    |   И ИсходящиеСообщения.КоличествоВыгрузок < 3";
Запрос.УстановитьПараметр("ОшибкаВыгрузки", Перечисления.кфкСтатусыСообщений.ОшибкаВыгрузки);
```

### «Залежалые» сообщения в статусе «Новое» больше N минут

Полезно для проверки SLA вне регламентного контроля интеграции:

```bsl
Запрос = Новый Запрос;
Запрос.Текст =
    "ВЫБРАТЬ
    |   ИсходящиеСообщения.Продюсер          КАК Продюсер,
    |   ИсходящиеСообщения.КлючРегистрации   КАК КлючРегистрации,
    |   ИсходящиеСообщения.ДатаРегистрации   КАК ДатаРегистрации
    |ИЗ
    |   РегистрСведений.кфкИсходящиеСообщения КАК ИсходящиеСообщения
    |ГДЕ
    |   ИсходящиеСообщения.Статус = &Новое
    |   И ИсходящиеСообщения.ДатаРегистрации < &Порог";
Запрос.УстановитьПараметр("Новое", Перечисления.кфкСтатусыСообщений.Новое);
Запрос.УстановитьПараметр("Порог", ТекущаяДатаСеанса() - 10 * 60); // 10 мин
```

### Размер очередей по топикам

```bsl
Запрос = Новый Запрос;
Запрос.Текст =
    "ВЫБРАТЬ
    |   Продюсеры.ИмяТопика          КАК Топик,
    |   КОЛИЧЕСТВО(*)                КАК Всего,
    |   СУММА(ВЫБОР КОГДА Исх.Статус = &Новое ТОГДА 1 ИНАЧЕ 0 КОНЕЦ) КАК Ожидают
    |ИЗ
    |   РегистрСведений.кфкИсходящиеСообщения КАК Исх
    |ЛЕВОЕ СОЕДИНЕНИЕ Справочник.кфкПродюсеры.ОбъектыМетаданных КАК Продюсеры
    |   ПО Исх.Продюсер = Продюсеры.Ссылка
    |СГРУППИРОВАТЬ ПО Продюсеры.ИмяТопика";
Запрос.УстановитьПараметр("Новое", Перечисления.кфкСтатусыСообщений.Новое);
```

!!! tip "Вывод в Telegram / на почту"
    Комбинируйте эти запросы с регламентным заданием прикладной конфигурации и отправляйте сводку в чат/письмо. Для алертов по SLA отдельный механизм не нужен — он [встроен](../configuration/alerts.md).
