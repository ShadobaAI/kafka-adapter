# Подключение в основную конфигурацию

Вариант для сильно кастомизированных решений, где адаптер является неотъемлемой частью прикладной конфигурации.

## Когда выбирать

- Конфигурация доработана и не обновляется из типовой поставки.
- Нужна глубокая интеграция адаптера в прикладную логику.
- Требуется полный контроль над точками встраивания.

## Плюсы и минусы

**Плюсы:**

- Полный контроль над точками встраивания.
- Возможность глубокой интеграции в прикладную логику.
- Дополнительные индексы СУБД создаются **автоматически**.

**Минусы:**

- Усложнение обновлений конфигурации.
- Требуется особое внимание при слиянии изменений (при каждом обновлении типовой).

## Форматы поставки

| Файл | Назначение |
|------|------------|
| `Adapter Kafka v*.cf` | Файл поставки конфигурации |
| `Adapter Kafka v* (CF).zip` | Проект EDT конфигурации — для разработки и доработки |
| `Adapter Kafka v*.clib` | [Библиотека расширения для EDT](https://its.1c.ru/db/edtdoc#content:10626:hdoc) — для подключения и обновления из библиотеки |

## Порядок подключения

=== "Конфигуратор"

    1. Получите файл поставки `Adapter Kafka v*.cf`.
    2. Откройте **Конфигурация → Поставка конфигурации**.
    3. Выберите файл `.cf`.
    4. Перенесите необходимые объекты в основную конфигурацию.
    5. Выполните действия из раздела [Адаптация подсистемы](adaptation.md).
    6. Обновите конфигурацию базы данных.

=== "EDT — из zip"

    1. Получите архив `Adapter Kafka v* (CF).zip`.
    2. Откройте проект в EDT.
    3. Выполните действия из раздела [Адаптация подсистемы](adaptation.md).
    4. Разверните проект на информационной базе.

=== "EDT — через библиотеку"

    1. Получите файл `Adapter Kafka v*.clib`.
    2. Создайте проект конфигурации на [поддержке библиотеки](https://its.1c.ru/db/edtdoc#content:10626:hdoc).
    3. Выполните действия из раздела [Адаптация подсистемы](adaptation.md).
    4. Разверните проект на информационной базе.
