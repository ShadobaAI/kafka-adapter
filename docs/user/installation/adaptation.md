# Настройка в EDT / Конфигураторе

Действия, которые необходимо выполнить на уровне метаданных — **один раз**, при первом подключении адаптера к целевой конфигурации.

## Определяемый тип `кфкПользователь`

В адаптере определён тип `кфкПользователь` для хранения ссылок на пользователей. По умолчанию он указывает на `СправочникСсылка.Пользователи` — служебный справочник из состава адаптера, предназначенный **только для разработки**.

!!! warning "Обязательно заменить"
    При внедрении в прикладную конфигурацию **замените** тип на `СправочникСсылка.Пользователи` **целевой** конфигурации (обычно он есть в БСП — `Справочник.Пользователи`).

### Как заменить

=== "EDT"

    1. Откройте дерево → **Общие → Определяемые типы → `кфкПользователь`**.
    2. Двойным щелчком откройте тип.
    3. В окне «Тип данных» укажите справочник пользователей целевой конфигурации (в типовых конфигурациях — обычно `СправочникСсылка.Пользователи`).
    4. Сохраните проект.
    5. Обновите конфигурацию базы данных.

=== "Конфигуратор"

    1. Откройте дерево конфигурации → **Общие → Определяемые типы → `кфкПользователь`**.
    2. Двойным щелчком откройте тип.
    3. В окне «Тип данных» укажите справочник пользователей целевой конфигурации (в типовых конфигурациях — обычно `СправочникСсылка.Пользователи`).
    4. Обновите конфигурацию базы данных.

## Оптимизация подписок на события

По умолчанию [подписки адаптера](../../project/modules.md#подписки-на-события) охватывают **все** ссылочные объекты и все наборы записей регистров — фильтрация по настройкам продюсера выполняется в обработчике.

!!! tip "Рекомендация"
    Чтобы избежать избыточных срабатываний подписок и повысить производительность, **сузьте состав источников** подписок под реальные потребности обмена:

    - **Удалите/не переносите** подписки для типов объектов, которые не будут участвовать в обмене. Например, если регистры расчёта не выгружаются — удалите подписку `кфкЗарегистрироватьИзмененияНабораРасчетаПриЗаписи`.
    - В подписке `кфкЗарегистрироватьИзмененияПриЗаписи` и `кфкЗарегистрироватьИзмененияПередУдалением` замените **«Любая ссылка»** на перечень конкретных ссылочных типов, которые реально участвуют в обмене.
    - В подписке `кфкЗарегистрироватьИзмененияНабораПриЗаписи` оставьте только нужные регистры (сведений/накопления/бухгалтерии) и уберите неиспользуемые — например, если выгружаются только регистры сведений, исключите регистры накопления и бухгалтерии.

    После сужения источников функциональность адаптера не меняется — объекты всё равно должны быть указаны в настройках [продюсера](../configuration/producers.md).
