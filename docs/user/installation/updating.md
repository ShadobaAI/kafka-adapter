# Обновление версий

Как обновить уже внедрённый адаптер на новую версию. Новые версии публикуются в [GitHub Releases](https://github.com/ShadobaAI/kafka-adapter/releases); состав файлов релиза — [Форматы поставки](../../project/delivery-formats.md).

## Перед обновлением — в любом варианте

1. **Прочитайте описание релиза** — там перечислены изменения и возможные особенности перехода.
2. **Остановите обмен**: на вкладке **Kafka / Администрирование / Фоновые задания** нажмите **Заблокировать** и дождитесь завершения активных потоков; отключите регламентное задание.
3. **Сделайте резервную копию** информационной базы.
4. Выберите момент, когда очереди максимально разобраны — так меньше сообщений «переживёт» обновление в промежуточных статусах.

## Вариант «расширение конфигурации»

=== "Конфигуратор"

    1. Получите новый файл `Adapter Kafka v*.cfe` из релиза.
    2. Откройте **Конфигурация → Расширения конфигурации**.
    3. Выберите расширение адаптера и загрузите новую версию: **Конфигурация → Загрузить конфигурацию из файла** (или удалите старое расширение и добавьте новое — настройки хранятся в данных базы и при этом не теряются).
    4. Обновите конфигурацию базы данных.
    5. Повторите шаги из [Адаптации подсистемы](adaptation.md), если они выполнялись (замена определяемого типа `кфкПользователь` в расширении).

=== "EDT — через библиотеку"

    Если проект расширения создан на **поддержке библиотеки** (`.xlib`), обновление сводится к обновлению библиотеки до новой версии и повторному развёртыванию проекта — ручное слияние не требуется. Подробнее: [1С:EDT — Работа с библиотеками](https://its.1c.ru/db/edtdoc#content:10626:hdoc).

## Вариант «основная конфигурация»

=== "Конфигуратор"

    1. Получите новый файл `Adapter Kafka v*.cf` из релиза.
    2. Выполните **Конфигурация → Сравнить, объединить с конфигурацией из файла**, отобрав объекты с префиксом `кфк`.
    3. Если объекты адаптера дорабатывались в вашей конфигурации — проверьте слияние по каждому изменённому объекту.
    4. Обновите конфигурацию базы данных.

=== "EDT — через библиотеку"

    Если конфигурация ведётся в EDT на поддержке библиотеки (`.clib`), обновите библиотеку до новой версии и разверните проект — изменения адаптера придут без ручного слияния.

## После обновления

1. Включите регламентное задание и снимите блокировку (**Разблокировать**).
2. Убедитесь, что подсистема включена: **Kafka / Администрирование**.
3. Проверьте, что очереди разбираются: статусы новых сообщений доходят до «Выгружено» / «Обработано» — см. [Мониторинг очередей](../operations/monitoring.md).
4. Просмотрите журнал регистрации на предмет ошибок фоновых заданий.

!!! info "Внешняя компонента обновляется вместе с подсистемой"
    Simple Kafka Connector 1C входит в состав поставки — отдельно обновлять его не нужно. Требуемая версия компоненты — **1.9.1+**.

!!! warning "Новые объекты и права"
    Если в новой версии появились объекты метаданных, роли `кфкБазовыеПрава` / `кфкПолныеПрава` обновятся вместе с подсистемой. Но если вы раздавали права на объекты адаптера собственными ролями — дополните их вручную. См. [Роли и права](roles.md).
