# Подключение как расширение конфигурации

:material-star: **Рекомендуемый способ** для типовых и часто обновляемых конфигураций.

## Когда выбирать

- Конфигурация типовая и регулярно обновляется из поставки.
- Нужно минимальное вмешательство в основную конфигурацию.
- Важно иметь возможность быстро отключить или удалить подсистему.

## Плюсы и минусы

**Плюсы:**

- Минимальное вмешательство в основную конфигурацию.
- Упрощённые обновления — расширение обновляется независимо.
- Возможность быстро отключить/удалить расширение.

**Минусы:**

- Ограничения механизма расширений 1С.
- Дополнительные индексы в СУБД [создаются вручную](https://its.1c.ru/db/v8326doc#bookmark:dev:TI000002802) (автоматически — только при лицензии «КОРП»).

## Форматы поставки

| Файл | Назначение |
|------|------------|
| `Adapter Kafka v*.cfe` | Файл расширения — подключается через «Расширения конфигурации» |
| `Adapter Kafka v* (CFE).zip` | Проект EDT расширения — для разработки и доработки |
| `Adapter Kafka v*.xlib` | [Библиотека расширения для EDT](https://its.1c.ru/db/edtdoc#content:10626:hdoc) — для подключения и обновления из библиотеки |

## Порядок подключения

=== "Конфигуратор"

    1. Получите файл поставки `Adapter Kafka v*.cfe`.
    2. Откройте **Конфигурация → Расширения конфигурации**.
    3. Нажмите **Добавить** и выберите файл `.cfe`.
    4. Обновите конфигурацию базы данных.
    5. Выполните действия из раздела [Адаптация подсистемы](adaptation.md) — замена определяемого типа `кфкПользователь`.

=== "EDT — из zip"

    1. Получите архив `Adapter Kafka v* (CFE).zip`.
    2. Откройте проект в EDT: **File → Import → General → Projects from Folder or Archive**.
    3. Выполните действия из раздела [Адаптация подсистемы](adaptation.md).
    4. Разверните проект на информационной базе.

=== "EDT — через библиотеку"

    1. Получите файл `Adapter Kafka v*.xlib`.
    2. Создайте проект расширения на [поддержке библиотеки](https://its.1c.ru/db/edtdoc#content:10626:hdoc).
    3. Выполните действия из раздела [Адаптация подсистемы](adaptation.md).
    4. Разверните проект на информационной базе.

---

После подключения перейдите к [Настройке подсистемы](../configuration/index.md) в режиме 1С:Предприятие.
