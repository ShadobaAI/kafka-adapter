# Установка и подключение

## Требования

- **Платформа 1С:Предприятие:** 8.3.21 и выше.
- **Библиотека стандартных подсистем (БСП):** 3.1.10 и выше — обязательное условие для конфигурации, в которую встраивается адаптер.
- **Kafka-инфраструктура:**
  - доступный кластер Apache Kafka;
  - настроенные топики для исходящих и входящих сообщений;
  - при необходимости — XDTO-пакет и модуль обмена КД 3.1.
- **Внешний сервис логирования** (опционально): любой HTTP/HTTPS-эндпоинт, принимающий JSON (Logstash, Elasticsearch, Loki).
- **Инструменты разработки:** EDT или Конфигуратор.

> Внешняя компонента для работы с Kafka встроена в подсистему. Отдельная установка и регистрация не требуются.

## Варианты подключения

Адаптер может быть подключён к прикладной конфигурации двумя способами.

<details>
<summary>Вариант 1. Расширение конфигурации — рекомендуется</summary>

Рекомендуемый способ для типовых и часто обновляемых конфигураций.

**Плюсы:** минимальное вмешательство в основную конфигурацию, упрощение обновлений, возможность быстро отключить расширение.
**Минусы:** дополнительные индексы в СУБД создаются вручную (при внедрении в конфигурацию с лицензией «КОРП» — создаются автоматически).

| Файл | Назначение |
|------|------------|
| `Adapter Kafka v*.cfe` | Файл расширения — подключается через «Расширения конфигурации» |
| `Adapter Kafka v* (CFE).zip` | Проект EDT расширения — для разработки и доработки |
| `Adapter Kafka v*.xlib` | [Библиотека расширения для EDT](https://its.1c.ru/db/edtdoc#content:10626:hdoc) — для подключения и обновления проекта из библиотеки |

**Порядок подключения:**
1. Получите файл поставки нужного формата.
2. Подключите расширение:
   - **Конфигуратор:** Конфигурация → Расширения конфигурации → Добавить → выберите `.cfe`.
   - **EDT:** откройте проект из `.zip` или создайте проект на поддержке библиотеки `.xlib`.
3. Обновите конфигурацию базы данных.

</details>

<details>
<summary>Вариант 2. Включение в основную конфигурацию</summary>

Подходит для сильно кастомизированных решений, где адаптер является неотъемлемой частью.

**Плюсы:** полный контроль над точками встраивания, возможность глубокой интеграции в прикладную логику.
**Минусы:** усложнение обновлений конфигурации, требуется особое внимание при слиянии изменений.

| Файл | Назначение |
|------|------------|
| `Adapter Kafka v*.cf` | Файл поставки конфигурации |
| `Adapter Kafka v* (CF).zip` | Проект EDT конфигурации — для разработки и доработки |
| `Adapter Kafka v*.clib` | [Библиотека расширения для EDT](https://its.1c.ru/db/edtdoc#content:10626:hdoc) — для подключения и обновления проекта из библиотеки |

**Порядок подключения:**
1. Получите файл поставки нужного формата.
2. Встройте объекты адаптера в основную конфигурацию через стандартный механизм поставки 1С:
   - **Конфигуратор:** Конфигурация → Поставка конфигурации → выберите `.cf`.
   - **EDT:** откройте проект из `.zip` или создайте проект на поддержке библиотеки `.clib`.
3. Обновите конфигурацию базы данных.

</details>

## Настройка в Конфигураторе / EDT

<details>
<summary>Определяемый тип кфкПользователь</summary>

В адаптере определён тип `кфкПользователь`, который используется для хранения ссылок на пользователей. По умолчанию он указывает на `СправочникСсылка.Пользователи` — справочник из состава адаптера, предназначенный только для разработки.

При внедрении в прикладную конфигурацию замените тип:

1. Откройте редактор определяемых типов (EDT или Конфигуратор).
2. Найдите тип `кфкПользователь`.
3. Установите тип значения: `СправочникСсылка.Пользователи` целевой конфигурации.
4. Обновите конфигурацию базы данных.

</details>
