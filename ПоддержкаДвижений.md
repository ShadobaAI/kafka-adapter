## Описание

Добавить поддержку выгрузки регистров подчинённых регистратору в Kafka. Единицей регистрации является регистратор, а не набор записей или отдельная запись.

Затрагивает следующие типы объектов метаданных 1С:
- `РегистрСведений` (режим записи — `ПодчинениеРегистратору`)
- `РегистрНакопления`
- `РегистрБухгалтерии`
- `РегистрРасчёта`

---

## Концепция

### Регистрация

При изменении набора записей регистрируется **регистратор** (документ), а не каждая запись отдельно.

Снимок данных — лёгкая структура с отбором по регистратору:

```json
{
  "Регистратор": "<ссылка на документ>"
}
```

### Сериализация

**Произвольный обработчик** — адаптер передаёт регистратор в метод сериализации, разработчик сам формирует сообщение.

**Конвертация данных (КД 3.1)** — адаптер выполняет запрос записей регистра по регистратору и перебирает выборку. Чтение набора через отбор не используется по соображениям производительности. Применяет правила к каждой записи, собирает массив объектов. Результат — **одно сообщение** в Kafka, тело которого содержит массив.

### Модель выгрузки

По умолчанию для всех регистров подчинённых регистратору используется **пакетная модель**: все записи выгружаются как массив в одном сообщении.

Если разработчику необходимо выгружать каждую запись как отдельный объект (**объектная модель**), он реализует это самостоятельно через программный API (см. ниже).

## Настройки продюсера

В поле **Тип объект** указывается полное имя объекта метаданных 1С, например:

| Тип объект | Описание |
|---|---|
| `РегистрНакопления.ТоварыНаСкладах` | Регистр накопления |
| `РегистрБухгалтерии.Хозрасчётный` | Регистр бухгалтерии |
| `РегистрРасчёта.Начисления` | Регистр расчёта |
| `РегистрСведений.ЦеныНоменклатуры` | Регистр сведений подчинённый регистратору |

> Регистр сведений с режимом записи `Независимый` регистрируется как обычный объект — существующий механизм, изменений не требует.
